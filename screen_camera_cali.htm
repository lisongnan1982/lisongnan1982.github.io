1.     Introduction

As illustrated in the figure below, the screen-camera calibration algorithm aims to find the 3D locations of the four screen corners in the camera coordinates. The calibration results can be used in many applications, such as holographic-like displaying; using the screen as a controllable planar light source for shape recovery; enabling the user's interaction with a specific region of the screen, etc.

Figure 1. Screen-camera calibration aims to find the 3D locations of the four screen corners in the camera coordinates.

 

2.     MATLAB code

We proposed a novel screen-camera calibration algorithm in:

 

[1] Songnan Li, King Ngi Ngan, Lu Sheng, ``Screen-camera calibration using a thread``, IEEE International Conference on Image Processing (ICIP2014), 27-30, Paris, 2014. [PDF]

 

Rather than using an external camera or a portable mirror like in previous studies, we use a more accessible and cheaper calibration objection, i.e., a thread. For technical details, please check our paper. The MATLAB implementation of [1] can be downloaded here. Below is a video demonstrating its usage (select 720p and watch it in fullscreen):

 

 

 

3.     Application demo

By using the proposed screen-camera calibration algorithm, we can get the 3D position of the screen in the camera coordinates. By using our 3D head tracking algorithm, we can get the 3D position of the user's viewpoint in the camera coordinates. Then, a holographic-like displaying system can be developed by locating the screen, the viewpoint, and the virtual CG object into the same world coordinates. There is a demo video:

 

 

Notice that the standard perspective projection API of OpenGL will not work for holographic-like displaying. We implemented the "generalized perspective projection" following the instructions of this paper:

[2] Robert Kooima, "Generalized perspective projection", online doc: http://csc.lsu.edu/~kooima/pdfs/gen-perspective.pdf
